# Contributing to GlowingEntities

## Mappings generation
The mappings file, located at `src/main/resources/fr/skytasul/glowingentities/mappings/spigot.txt`, contains NMS names mapping to obfuscated names.

It is a custom file type that contains ONLY the necessary mappings for GlowingEntities, to keep the file short, and that contains all versions in one single file.

The file is autogenerated by [reflection-mappings-shrieker](https://github.com/SkytAsul/ReflectionTools).

To trigger the generation, either launch the `src/mappingsgen/java/fr/skytasul/glowingentities/GlowingEntitiesMappingsGenerator.java` Java file directly in your IDE, or use this maven command:

```sh
mvn build-helper:add-source -Dsources="src/mappingsgen" exec:java -Dexec.mainClass="fr.skytasul.glowingentities.GlowingEntitiesMappingsGenerator" -Dexec.classpathScope="compile"
```

## Adding a new version
First, add the new version to the list of mappings to autogenerate in the `src/mappingsgen/java/fr/skytasul/glowingentities/GlowingEntitiesMappingsGenerator.java` file.

Then, launch the mappings generation as explained above. If it fails when generating your new version, then probably it means that the internals have changed and our reflection code is no longer compatible. You'll have to adapt it in the `GlowingEntities.Packets#loadReflection` method.
